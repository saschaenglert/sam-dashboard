Next step - Q2 2026
Separate
the agents.
Each agent needs its own brain. Right now they all share one. That limits how far I can push them.

What I'm building next

Right now every agent lives under one shared CLAUDE.md. That was fast to build but it creates leaks - context from one agent bleeds into another, and there's no clear boundary between what each one knows or is allowed to do.

The next iteration: every agent gets its own markdown brain - a dedicated folder with soul.md (role and boundaries), identity.md (name, tone), user.md (who it serves), heartbeat.md (30-minute checklist), and memory.md (what it has learned). Proper separation of concerns.

Why this matters

Agents don't scale because of bigger models. They scale because of sharper boundaries. An agent that knows exactly what it owns, what it can do, and who it answers to - is dramatically more useful than a general-purpose one sharing everything.

This pattern was crystallized by Nate B Jones in the video on the right. I had already been moving in this direction - the video gave me the vocabulary.

The plan

1. Build a per-agent folder template (soul, identity, user, heartbeat, memory).
2. Move each existing agent into its own scoped workspace.
3. Add per-agent autonomy levels - not one global L1-L4 but one per agent.
4. Introduce an orchestrator agent that delegates tasks, instead of a master CLAUDE.md that tries to know everything.
5. Split memory in Supabase so each agent has its own scope - no cross-contamination.

After that

With separation in place, I can run specialist agents: a sales agent that only handles outreach, a research agent that only finds and scores opportunities, a content agent that only writes. Each one deeper, each one safer to give more autonomy.
